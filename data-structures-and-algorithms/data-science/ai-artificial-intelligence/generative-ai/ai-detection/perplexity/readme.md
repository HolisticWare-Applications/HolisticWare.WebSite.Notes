Calculating perplexity in Python is a statistical method to detect AI-generated content by measuring how "surprised" a language model is by a given text. AI-generated text typically has low perplexity (predictable, low surprise) because it follows the probabilistic patterns the model was trained on, while human writing has high perplexity. 

How Perplexity Detects AI Content

Low Perplexity = AI: The model easily predicts the next word, meaning the text is likely generated by a similar model.
High Perplexity = Human: The text is unpredictable, creative, or messy, which models find "surprising".
Burstiness: A complementary metric, measuring the variance in sentence structure. Human text has high burstiness (mixing long/short sentences), while AI is uniform.

import torch
from transformers import GPT2LMHeadModel, GPT2Tokenizer
import numpy as np

def calculate_perplexity(text, model_id='gpt2'):
    # Load pre-trained model and tokenizer
    tokenizer = GPT2Tokenizer.from_pretrained(model_id)
    model = GPT2LMHeadModel.from_pretrained(model_id)
    
    # Encode the text
    inputs = tokenizer(text, return_tensors='pt')
    input_ids = inputs['input_ids']
    
    # Calculate loss (negative log-likelihood)
    with torch.no_grad():
        outputs = model(input_ids, labels=input_ids)
        loss = outputs.loss
        
    # Perplexity is the exponent of the loss
    perplexity = torch.exp(loss)
    return perplexity.item()

# Example Usage
ai_text = "The quick brown fox jumps over the lazy dog."
human_text = "As I was walking to the store, I saw a peculiar cat wearing a tiny top hat."

print(f"AI Text Perplexity: {calculate_perplexity(ai_text)}")
print(f"Human Text Perplexity: {calculate_perplexity(human_text)}")
# Typically, lower numbers = more likely to be AI.

Key Considerations for AI Detection
Model Selection: While GPT-2 is fast, more complex models (e.g., GPT-3, GPT-4) require larger models to achieve accurate perplexity measurements.
