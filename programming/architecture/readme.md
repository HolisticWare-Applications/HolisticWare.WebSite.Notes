    *   CQRS CQRS (Command Query Responsibility Segregation)
    
        *   design pattern that separates data writing (Commands) from data reading (Queries) into distinct models, improving performance, scalability, and maintainability for complex systems by optimizing each side independently, often using separate databases or data structures. It tackles systems where read/write needs diverge, allowing the command side to handle state changes (create, update, delete) and the query side to efficiently serve display/reporting needs, sometimes with event sourcing for auditin
